I am sending this email to the list so that we have a record of the  
formatting issues, but I think these are low priority.

When UseOA-ROSE is compiled with gcc 4.0.2, it probably links with a  
different gcc library.  As such some of the formatting appears to  
change.  Also, there appears to be some minor ordering changes.

1) The "this" location and "method" location are swapped in the output.

Testing TestCases/AliasTests/invokeBaseCtor.C, TestResults/ 
AliasIRInterface/invokeBaseCtor.C.out checked by BSW 6/26/06

<     LOCATION = { < SymHandle 
("method:SubClass::SubClass__SubClass__scope__SubClass____MFb_SubClass__ 
Fb_v_Gb___Rb__Base__Re___Fe_"), not local > }
28a28
 >     LOCATION = { < SymHandle 
("method:SubClass::SubClass__SubClass__scope__SubClass____MFb_SubClass__ 
Fb_v_Gb___Rb__Base__Re___Fe_"), not local > }
63a64
 >     LOCATION = { < SymHandle 
("this::SubSubClass::SubSubClass__SubSubClass__scope__SubSubClass____MFb 
_SubSubClass__Fb_v_Gb___Rb__Base__Re____sep____Rb__i__Re___Fe_"),  
local > }
66d66
<     LOCATION = { < SymHandle 
("this::SubSubClass::SubSubClass__SubSubClass__scope__SubSubClass____MFb 
_SubSubClass__Fb_v_Gb___Rb__Base__Re____sep____Rb__i__Re___Fe_"),  
local > }


2) Floating point numbers are formatted to a different length.
