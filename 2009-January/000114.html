On Jan 20, 2009, at 3:25 AM, Simone Pellegrini wrote:

> Dear Michelle,
> my name is Simone Pellegrini and I am a PhD student from the  
> university of Innsbruck. My research topic is related to parallel  
> programs optimization, practically we want to optimize existing  
> parallel programs by applying some well known transformations (like  
> MPI statements movement... optimization of MPI collective  
> operations, etc...) in a fully automatic way.
>
> In order to do that, we need to perform static program analysis in  
> oder to see where the dependencies are and how to transform the  
> original code without any change to the semantics. The research I  
> have done about source-to-source transformations... and program  
> analysis bring me to these 2 tools, ROSE and OpenAnalysis. In the  
> Rose mailing list I've asked more details about the Rose to OA  
> integration and they gave me a pointer to you. Can you give me some  
> pointers about this integration? Is it working? or just a work-in- 
> progress!?

At this point I would categorize the connection between OA and ROSE  
(UseOA-ROSE) as a work in progress.

> This is a very important step for us, we have to choose a tool and  
> this will influence all our research and our results. If we choose a  
> transformation tool which is not able to parse most of the existing  
> production source codes... our work will be worth nothing. Actually  
> the integrated program analysis implemented in Rose is not so good  
> (a lot of bugs) and OA seems to be better (on the paper)... but I  
> have no idea about how much stable it is... and if it can be used  
> with very complex code or use small code pieces.

Yes it is difficult to find a good tool.  If you want a C++ source-to- 
source tool that can handle most of C++ then ROSE might be your best  
bet.  The analysis piece is still being developed, therefore it  
probably is not as robust as you would like right now.

On Jan 14, 2009, at 3:25 AM, Simone Pellegrini wrote:

> We need a frontend capable to parse C/C++ and Fortran source files. We
> looked inside ROSE... but we are encountering a lot of problems with  
> the
> integrated analysis (bugs... slow...). I saw you claims that OA can be
> used with ROSE, where can I find informations about this?  
> Furthermore we
> are considering other ways... we also looked inside the LLVM project,
> actually the Clang compiler, but as far it is a compiler, clang is not
> meant to be used as a source-to-source compiler. Probably the best
> option would be to work with the LLVM intermediate representation  
> and I
> have heard that OA can be used also with the LLVM Bytecode. How can I
> get more details about the current status of integration?


OA could hook up to LLVM but it doesn't.  Analysis in general should  
be easier in LLVM because the representation is lower-level  
(essentially typed 3-address code).  It might be difficult to do  
source-to-source compilation with LLVM.

If you find a tool that satisfies your requirements fully, please let  
me know.  =)
