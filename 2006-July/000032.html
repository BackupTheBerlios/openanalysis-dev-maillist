Andy, Michelle, and others:

Unless you have strong(er) motivation for doing so, I would suggest not
migrating to 0.8.8a.  In trying to fix the varargs bug, I encountered a
bug related to varargs and 0.8.8a (but not 0.8.7a).  I have sent this to
Dan, but don't expect a fix any time soon.  This caused one of our
benchmarks to abort.

I imagine the bug occurs for a program like:

int *returnSometing() { return; }

If I remember correctly, when deciding whether the return statement
is a pointer assignment statement, I look at the return type of the
function.  This would label this return as a pointer assignment statement,
though it obviously is not.  There isn't too much point in speculating, I
will look at this soon.  But I suspect the bug is easy to fix.

Brian
