Dear all,
I installed OpenAnalysis on my machine and I got a compilation error 
during the make procedure.

I am using the latest Gcc 4.3.2 and the problem is within the file: 
src/OpenAnalysis/Utils/Util.hpp

The error is the following:
DataFlow/CallGraphDFSolver.cpp: In constructor 
‘OA::DataFlow::CallGraphDFSolver::CallGraphDFSolver(OA::DataFlow::CallGraphDFSolver::CallGraphDirectionType, 
OA::DataFlow::CallGraphDFProblem&)’:
DataFlow/CallGraphDFSolver.cpp:29: error: ‘getenv’ was not declared in 
this scope
DataFlow/CallGraphDFSolver.cpp:29: warning: deprecated conversion from 
string constant to ‘char*’
make[2]: *** [libOAul_a-CallGraphDFSolver.o] Error 1
make[2]: Leaving directory 
`/home/spellegrini/Desktop/OpenAnalysis-0.3a1/src/OpenAnalysis'
make[1]: *** [all-recursive] Error 1
make[1]: Leaving directory 
`/home/spellegrini/Desktop/OpenAnalysis-0.3a1/src'
make: *** [all-recursive] Error 1

Actually the error is simple solvable because what you need is to 
insclude the cstdlib libreaty in the Util.hpp file:
...
#ifndef OA_Util_H
#define OA_Util_H

#include <cstdlib>
#include <string>

// Could use templates within OA namespace
...

and this fixes the problem in my configuration.

regards Simone Pellegrini
