Hi all,

- Resolved bug #8068, "GenOutputTool chokes on OA_ptr<std::set<IRHandle> >"
      GenOutputTool will now gracefully handle containers that are wrapped in a
      single OA_ptr.

This means GenOutputTool should now be able to generate the last two
maps in  DataFlow/ParamBindings.hpp.

GenOutputTool is still unable to generate code for type-defined types,
but you can hack around this by substituting all typedef'd types with
whatever they alias.

Thanks,
