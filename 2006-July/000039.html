Andy,

WN* is a pointer to a Whirl Node, which is conceptually similar to  
SgNode*.
ST* is a pointer to a symbol table entry.

In createAndMapNamedRef, the wn parameter is the MemRefHandle we are  
constructing a MemRefExpr for.  The st parameter is what we turn into  
a SymHandle.

Note that you don't need to do any of that fully_qualified_name stuff  
in UseOA-ROSE.  That is a hack to deal with Open64 weirdness.

Just so everyone is on the same page, Andy is doing a refactor of the  
findAllMemRefExprs functionality in UseOA-ROSE to match the refactor  
I did in UseOA-Open64.
