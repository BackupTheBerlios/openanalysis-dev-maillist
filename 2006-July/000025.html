If you all recall, Brian was suggesting that people cleanup the  
config/Makefile.inc file in UseOA-ROSE.  I changed things so that we  
won't be using the Makefile.inc in the same way anymore.  Now you  
need to write your own script, that you do not check into the  
repository and source that before compiling.  This way things will be  
similar to how things work in UseOA-Open64.  Here is my script as an  
example:

#### User-specific configuration for UseOA-ROSE

# Specifies what debugging & profile flags are sent to the compiler
# and linker
#PROFILE_OPTIONS = -O0 -g -pg
setenv PROFILE_OPTIONS '-O0 -g'
#PROFILE_OPTIONS =

# Top-level UseOA source directory
setenv topsrcdir /s/parsons/c/fac/mstrout/OA/armstrong/UseOA-ROSE

# Root of ROSE installation
setenv ROSE_INST_ROOT /s/bach/e/proj/oadev/ROSE/rose-0.8.7a/armstrong

# OpenAnalysis variables
setenv OPENANALYSIS_DIR /s/parsons/c/fac/mstrout/OA/armstrong/ 
OpenAnalysis

setenv OPENANALYSIS_ARCH i686-Linux
#OPENANALYSIS_ARCH=opteron-Linux

# Compiler/linker
setenv CXX g++
setenv CXXLD g++
setenv CXXFLAGS '-g -O0'
