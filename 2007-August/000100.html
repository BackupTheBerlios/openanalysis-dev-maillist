    At this point,  We create ExprTree's on the fly whenever we
    need ExprTree for the given statement.  We do not store
    information about statement and its ExprTrees.

    I think your suggesion to get an Iterator over ExprTree for a statement
    indirectly suggests that we need a map to store
    ExprTrees for each statement,  therefore, we need to create ExprTrees
    only once.

    How about creating a map <Stmt, Set<ExprTrees> > at the same time
    we create MemRefExprs for the whirl node ?
